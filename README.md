Software Configuration Management (SCM) is a discipline in software engineering that systematically 
manages and controls changes to software throughout its development lifecycle. It ensures that software 
products are built correctly, changes are tracked, and every version of the system is identifiable and 
reproducible. 
 
This project applies SCM practices to a Web-Based Library Management System (LMS) — a real-time 
application developed to automate and streamline library operations in an institution. The system handles 
book cataloguing, member management, issue/return tracking, fine calculation for overdue books, and 
generates detailed transaction reports for each member. 
 
Git is used as the version control system and GitHub is used as the remote repository hosting platform. All 
SCM activities including version control, change management, configuration auditing, and release 
management are performed using Git and GitHub throughout the project lifecycle. 
 
The project was developed in two major versions: 
• v1.0.0 — Basic library system with book cataloguing, member registration, and manual issue/return 
tracking 
• v2.0.0 — Real-time library system with MemberID-based lookup, automated fine calculation, and 
digital receipt/report generation
